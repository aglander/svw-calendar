## Calendar import pipeline

The app loads calendar data from [`public/data/events.json`](public/data/events.json), which is generated by an hourly import job.

### Configure sources

Edit [`config/calendar-sources.public.json`](config/calendar-sources.public.json):

- one entry per `teamId`
- set `enabled: true` and provide a `url` (`webcal://` or `https://`)
- optional `venueDefault` (`Sportplatz`, `MZH`, `Turnhalle`)

### Run importer locally

```sh
npm run import:calendars
```

This writes:

- `public/data/events.json`
- `public/data/last-sync.json`

### Optional: import operator shadow plan (Excel)

1. Place your latest Excel file at `data/operator-plan.xlsx`.
2. Run:

```sh
npm run import:shadow
npm run import:calendars
```

This writes:

- `public/data/shadow-slots.json`
- `public/data/shadow-summary.json`

Then calendar events are enriched with `verification` status based on overlapping weekly shadow slots.

### Calendar modes in UI

- `Standard` (light): shows imported calendar events from `events.json`.
- `Plan` (dark): shows recurring shadow slots from `shadow-slots.json` (SVW, Schule, extern).
- `Tanzraum` is available as 4th venue across filters and day view.

### GitHub Action

The workflow at [`.github/workflows/import-calendars.yml`](.github/workflows/import-calendars.yml) runs:

- every hour (`0 * * * *`)
- manually via `workflow_dispatch`

If output files changed, it commits and pushes updated data.
